Use this page when you want to call an IOVI service from the outside. The current GTM surface is public and split across four live services: Payment SL, Generic Verifier, Bundler/Marketplace, and Base-Layer API.

## Try The Public Services

Primary Payment SL sandbox:

```text
https://eon-payment-sl-demo-production.up.railway.app
```

First check:

```bash
curl https://eon-payment-sl-demo-production.up.railway.app/health
```

## First Payment Flow

The public sandbox is stateful, so treat it as a shared demo service. Use these requests as the minimal integration shape, then switch to the rendered reference for exact request and response schemas.

```bash
BASE=https://eon-payment-sl-demo-production.up.railway.app

curl "$BASE/health"

curl -X POST "$BASE/operator/init" \
  -H "Content-Type: application/json" \
  -d '{"issuer_vk":"circle_inc_verification_key","reset_existing":false}'

curl -X POST "$BASE/wallets" \
  -H "Content-Type: application/json" \
  -d '{"label":"Alice","vk":"alice_vk","kind":"user"}'
```

Continue with:

```text
POST /actions/mint
POST /operator/batch
POST /verifier/accept-latest-batch
GET /balances/{address}
```

Verifier reads are the accepted state surface. Operator state is useful for batching and debugging, but it is not final wallet state.

## Trust And Auth Notes

- The Payment SL URL is a sandbox, not a production custody service.
- Public read endpoints are intended for GTM demos and integration discovery; write endpoints may mutate shared sandbox state or require an API key.
- Current playground requests may submit raw VK-like values; production flows should use signed intents or stronger auth.
- Scalar encoding means payload bytes can be represented as UTXO `Data`; it is not proof of base-layer inclusion.
- Bundles transport and order child payloads; child semantic layers still own validity.
- Base-layer APIs do not interpret semantic-layer meaning.
